# 📌 Telegram Session Stealer — Полный разбор (для архива)

> Исследовательская заметка  
> Цель: понять механизм кражи сессии и способы защиты  
> Контекст: тестирование собственной реализации мессенджера  

---

## 1. Что такое Session Stealer

Session Stealer — это тип вредоносного ПО, которое не подбирает пароль и не ломает 2FA, а **крадет уже активную сессию пользователя**.

В случае с :contentReference[oaicite:0]{index=0} Desktop сессия хранится локально в папке:
Windows:  
%AppData%\Telegram Desktop\tdata

macOS:  
~/Library/Application Support/Telegram Desktop/tdata


Если злоумышленник получает эту папку — он получает аккаунт.  
Без SMS. Без кода. Без 2FA. Просто заходит.

# 2. Полный цикл атаки (от А до Я)

## Этап 1 — Вектор заражения

Самые частые способы:

### 🔹 1. Фишинг
- Поддельная страница входа
- "Голосование", "Telegram Premium", "Срочное сообщение"
- Жертва сама вводит номер + код

### 🔹 2. Социальная инженерия
- Сообщение от знакомого
- Бот "Перенос чата"
- Просьба ввести код подтверждения

### 🔹 3. Троян / Инфостилер
- Файл .exe, .lnk, .cmd
- "Полезная программа"
- Пакет Python / npm
- "Модифицированный Telegram"

Это самый опасный сценарий.

---

## Этап 2 — Поиск сессии

Типичный алгоритм инфостилера:

```pseudo
locate %AppData%\Telegram Desktop\tdata
if Telegram.exe running:
    kill process
copy folder tdata
zip archive
send to attacker
restart Telegram
```
Некоторые стилеры:

- Архивируют папку
    
- Ставят пароль на архив
    
- Отправляют через Telegram Bot API
    
- Или через HTTP POST на сервер
    

Пользователь ничего не замечает.

---

## Этап 3 — Использование украденной сессии

Атакующий:

1. Копирует `tdata`
    
2. Вставляет в свою папку Telegram Desktop
    
3. Запускает клиент
    

Аккаунт открывается автоматически.

⚠️ Telegram считает это продолжением существующей сессии, а не новым входом.

---

# 3. Почему 2FA не спасает

2FA работает на этапе входа.

Но если украден:

- session key
    
- auth token
    
- локальный ключ шифрования
    

то вход уже не требуется.

Некоторые стилеры дополнительно ищут:
2FA.txt
# 4. Технические детали хранения

### В папке `tdata` хранятся:

- Ключи сессии
    
- Кэш сообщений
    
- Локальные настройки
    
- Данные аккаунта
    

Сессионные ключи:

- Не имеют строгого срока жизни
    
- Действуют до ручного выхода

# 5. Продвинутые техники

## 🔸 Завершение процесса

Стилер убивает Telegram перед копированием — чтобы снять lock с файлов.

## 🔸 Архивация

Создается `MT.zip` или аналогичный архив.

## 🔸 Экcфильтрация

Данные отправляются:

- Через Telegram Bot
    
- Через Discord webhook
    
- Через HTTPS
    

## 🔸 Отключение уведомлений

Иногда атакующий не завершает старую сессию, чтобы жертва не заметила вход.

# 7. Защита пользователя

## 🔐 1. Включить 2FA

Да, не идеал, но повышает барьер.

## 🔒 2. Включить Local Passcode

Шифрует часть локальных данных.

## 👀 3. Проверять активные сессии

Settings → Devices → Active Sessions

## 🛡 4. Антивирус

Современные EDR способны выявлять стилеры  
Отчёты по подобным атакам регулярно публикует Kaspersky.

## 🧠 5. Поведенческая гигиена

- Не скачивать сомнительные файлы
    
- Не вводить коды где попало
    
- Не использовать сторонние клиенты

# 8. Защита на уровне своей системы (для тестирования)

Если ты разрабатываешь аналог Telegram:

## 🔹 1. Никогда не хранить session-token в открытом виде

Использовать:

- OS keychain
    
- Secure Enclave
    
- DPAPI (Windows)
    

## 🔹 2. Привязывать сессию к устройству

- Hardware fingerprint
    
- Device binding
    
- IP anomaly detection
    

## 🔹 3. Ограниченный TTL токена

Не "вечная" сессия, а:

7–30 дней максимум

## 🔹 4. Ротация ключей

Периодическая регенерация session keys.

## 🔹 5. Шифрование + защита от копирования

- Ключ, связанный с machine ID
    
- Без него расшифровка невозможна
    

## 🔹 6. Детектирование аномалий

- Резкая смена геолокации
    
- Одновременный вход с двух устройств
    
- Подозрительный User-Agent

# 9. Архитектурный вывод

Главная уязвимость — **локальное хранение сессии в копируемом виде**.

Если токен можно:

- Скопировать
    
- Перенести
    
- Использовать без повторной аутентификации
    

— значит его украдут.

Это не вопрос «если», это вопрос «когда».

---

# 10. Ключевой принцип безопасности

Пароль можно защитить.  
SMS можно защитить.  
Но если токен — это "золотой билет",  
его нужно:

- шифровать
    
- ограничивать по времени
    
- привязывать к устройству
    
- отслеживать аномалии
