# 🧟‍♂️ Что такое легаси-проект

> _"Legacy code is code without tests." — Michael Feathers_

## 🏯 Определение

**Легаси-проект** (*legacy project*) — это код, который:
- Был написан давно или по устаревшим стандартам
- Использует старые технологии или библиотеки
- Плохо задокументирован
- Сложен для поддержки и изменений
- Часто "работает — не трогай"

---

## 🧨 Приметы легаси-кода

- Использование `var` вместо `let/const`
- Классовые компоненты в React 18+
- jQuery + React вместе (!)
- Отсутствие модульной структуры
- Много `console.log(...)`
- Нет тестов
- Огромные функции по 200 строк
- Зависимости, которые не обновлялись годами
- `// TODO: refactor` из 2017

---

## 🪤 Почему такие проекты не переписывают?

- 💸 **Бюджет** — переписать всё = дорого
- 😱 **Страх** — никто не помнит, как оно работает
- 📈 **Бизнес** — "нам нужно фичи, а не рефакторинг"
- 💥 **Риск** — любое изменение может всё сломать

---

## 🧘 Как работать с легаси-кодом

1. Не паникуй 😮‍💨
2. Изучи архитектуру, найди точки входа
3. Пиши **тесты вокруг** — до рефакторинга
4. Делай **маленькие изменения**
5. Мигрируй по частям (например, классовые → функциональные компоненты)
6. Документируй и улучшай по ходу

---

## 🧾 Чеклист: как узнать, что ты в легаси-проекте

- [ ] Ты боишься трогать код
- [ ] Никто не знает, кто писал этот файл
- [ ] Изменение одной строки ломает всё
- [ ] Нельзя обновить библиотеки без боли
- [ ] Всё ещё работает, но никто не знает почему

---

## 💬 Цитаты

> "Если это работает — не трогай. Если не работает — открой консоль."  
> — Каждый разработчик легаси

> "Лучше медленно переписывать, чем быстро чинить навсегда."  
> — Кто-то уставший

---

## 🧙 Советы для джедая легаси-кода

- Пиши новые фичи как **отдельные компоненты**
- Не пытайся сразу всё переделать — **итерации рулез**
- Если проект на React — **переводи классы в хуки**
- Всегда пиши комментарии к тому, что сам не понял сразу
