# **Function Component** — современный стиль
📌 Пример:
```js
function solom({name}) {
	return <h1>Salom {name}!</h1>;
}
```
Сейчас это + **Hooks** — стандарт разработки.
### ✅ Плюсы:
- Меньше кода, проще читать.
- Поддержка **Hooks** (`useState`, `useEffect`, и вся магия).
- Легче тестировать и рефакторить.
- Быстрее рендерятся (чуть-чуть).
- Совместимы с **React Server Components**.
## 🏯 **Class Component** — старый самурай
📌 Пример:
```js
class Salom extends React.Component() {
	render() {
		return <h1>Salom {name}!</h1>;
	}
}
```
### 🔻 Минусы
- Много шаблонного кода (`this`, `constructor`, `bind`, `super`).
- Нет доступа к Hooks (только через обёртки или сторонние хаки).
- Труднее читать и сопровождать.
---
### 🤔 **Когда использовать какой?**
| Ситуация                                            | Что выбрать                                    |
| --------------------------------------------------- | ---------------------------------------------- |
| Новый проект / свежий компонент                     | ✅ **Function**                                 |
| Нужно использовать `useState`, `useEffect` и прочие | ✅ **Function**                                 |
| Работаешь с кодом до 2019 года                      | 👴 **Class** (временно)                        |
| Поддержка старого кода                              | 🔧 **Class**, но можно постепенно переписывать |
|                                                     |                                                |

## 🧪 Бонус: Фичи, которых нет в классовых компонентах

- `useReducer` — мини Redux прямо внутри
- `useContext`, `useMemo`, `useCallback`, `useTransition` (React 18)
- `custom hooks` — мощнейшая фишка
- React Server Components
---
## 🧘 Итого

> **Если ты не на работаешь в легаси-проекте — забудь классы.**  
> Пиши функционально, используй Hooks, и твой код будет как катана: лёгкий, быстрый и смертельно точный ⚔️
